# Contributing

WaveSentinel accepts defensive-only contributions.

Read [CONTRIBUTOR.md](CONTRIBUTOR.md) first for project ownership, scope, and contribution boundaries.

## Rules

- Do not add attack automation.
- Do not add deauthentication execution.
- Keep all changes limited to detection, logging, visualization, documentation, or validation tooling.
- Test with a real monitor-mode adapter before opening a pull request.
- Update documentation when runtime behavior, CLI usage, or dashboard behavior changes.
